## Objetivo

Añadir al editor una herramienta visual para pintar `mapa.pisable` por celdas, usando la convención runtime `#` = pisable y `.` = bloqueado, y serializando siempre un string multiline `rows × cols`.

## Contexto

El runtime táctico ya consume `mapa.pisable` y valida movimiento sobre esa máscara, pero los mapas autorados quedan con todo `#`. El editor ya tiene calibración visual (`CalibradorGridDialog.jsx`) con imagen, overlay y campos `tile_w`, `tile_h`, `cols`, `rows`, `origen_px`; el pincel debe usar esa calibración existente sin recalibrarla.

## Ficheros (qué SÍ tocar)

- `src/components/aventura/WalkmaskBrushDialog.jsx` — nuevo diálogo visual con imagen, grid, overlay rojo de celdas bloqueadas, modos `Pisable`/`Bloqueado`, pintar con click/drag, cancelar/aplicar.
- `src/components/aventura/SeccionLocalizaciones.jsx` — botón “Pintar walkmask”, estado abrir/cerrar, y aplicación de `mapa.pisable`.
- `docs/specs/F3.2_pincel_walkmask.md` — esta mini-SPEC.

## Anti-patrones (qué NO)

- No tocar `dm_virtual`, backend, YAML ni assets.
- No cambiar la convención: `#` pisable, `.` bloqueado.
- No recalibrar `tile_w`, `tile_h`, `cols`, `rows`, `origen_px`.
- No instalar dependencias.
- No rediseñar la sección de mapas ni extraer utilidades grandes.

## Comportamiento

- Si `mapa.pisable` falta o no encaja con `cols × rows`, inicializar internamente todo a `#`.
- Al aplicar, escribir `mapa.pisable` como:
  - exactamente `rows` líneas;
  - cada línea de longitud `cols`;
  - solo `#` o `.`.
- Click y drag con botón pulsado pintan la celda bajo cursor.
- No se pinta fuera de tablero.
- El botón “Rellenar todo pisable” puede resetear la máscara a todo `#`.
